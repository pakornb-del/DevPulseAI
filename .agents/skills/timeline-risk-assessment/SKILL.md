# Skill: Timeline Risk Assessment

ทักษะนี้มีไว้สำหรับวิเคราะห์ความเสี่ยงและแจ้งเตือนเมื่อระบบมีแนวโน้มที่จะส่งมอบไม่ทันกำหนดเดดไลน์:

## 1. การคำนวณและประเมินผล
* ดึงข้อมูล % ความคืบหน้า จากคอลัมน์ N ของชีต `Project` ของทีมที่เกี่ยวข้อง
* เปรียบเทียบวันปัจจุบันกับวันที่ในคอลัมน์ J (กำหนดส่งงาน)
* **เกณฑ์ระดับความเสี่ยง (Risk Assessment Matrix):**
  * **ความเสี่ยงสูง (High Risk):** ความคืบหน้ายังน้อย (เช่น < 50%) แต่ระยะเวลาก่อนถึงเดดไลน์เหลือน้อยกว่า 10% หรือเกินเดดไลน์แล้วแต่สถานะยังไม่เป็น `Done`
  * **ความเสี่ยงปานกลาง (Medium Risk):** สถานะงานติดขัด หรือมีหมายเหตุระบุอุปสรรค เช่น รอการทดสอบ หรือติดปัญหาประปราย
  * **เสร็จทันกำหนด (On Track):** ความคืบหน้าเหมาะสมตามสัดส่วนเวลา และไม่มีหมายเหตุแจ้งปัญหาคอขวด

## 2. การวิเคราะห์หมายเหตุสำคัญ (Remarks Semantic Analysis)
* เอเจนต์ต้องอ่านและตีความข้อมูลเชิงคุณภาพจากคอลัมน์หมายเหตุ (คอลัมน์ P ในชีต Project หรือหมายเหตุในชีต ST*)
* ตรวจจับอุปสรรคเชิงระบบ เช่น:
  * *"น้ำท่วมใต้"* $\rightarrow$ ถือเป็นปัจจัยภัยธรรมชาติจากภายนอกที่กระทบกำหนดการส่งมอบ (เช่น เลื่อนวัน TCAS69)
  * *"รอฝั่งจัดซื้ออนุมัติ"* $\rightarrow$ ถือเป็นความเสี่ยงจากขบวนการดำเนินงานขององค์กร
  * *"ย้ายไปปีงบ 70"* $\rightarrow$ ถือเป็นงานที่ได้รับการปรับเลื่อนลำดับความสำคัญ (เปลี่ยนสถานะเป็น Paused)

## 3. ตัวอย่างตรรกะการวิเคราะห์ (Scenario Example)
* เมื่อได้รับคำถาม: *"ระบบบรรจุ ทำไปแล้วกี่เปอร์เซ็นต์ คิดว่าจะทำเสร็จทันก่อนส่งงานหรือไม่"*
  1. ค้นหาระบบชื่อ "ระบบบรรจุ" ข้ามทุกชีต `Project` ของ 7 ทีม
  2. พบว่าระบบบรรจุอยู่ในไฟล์ของ **ทีมพี่ไก่ (จิราภา)** ในชีต **ST2**
  3. ดึงค่าคอลัมน์ N (เปอร์เซ็นต์ความคืบหน้า) พบว่าเป็น 75%
  4. ดึงคอลัมน์ P (หมายเหตุ) พบว่าระบุว่า *"ทดสอบเสร็จสิ้นภายในสัปดาห์นี้"*
  5. สรุปวิเคราะห์และแจ้งเตือน: *"ระบบบรรจุ คืบหน้า 75% คาดว่าจะทดสอบเสร็จทันภายในสัปดาห์นี้ ประเมินความเสี่ยงต่ำ คาดว่าจะเสร็จทันส่งงานแน่นอน"*
