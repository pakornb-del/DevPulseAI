# Skill: Resource Optimization

ทักษะนี้ช่วยให้หัวหน้าฝ่ายพัฒนาระบบสารสนเทศสามารถบริหารคิวงานของลูกน้องทั้ง 16 คนได้อย่างเหมาะสม:

## 1. การตรวจสอบสถานะงานรายบุคคล (Developer Task Tracking)
* ค้นหารายชื่อพนักงานในทุกชีตรหัส `ST*` ของทุกไฟล์ เพื่อหาภาระงานย่อยทั้งหมด
* คัดกรองเฉพาะแถวงานที่แมปกับชื่อผู้รับผิดชอบที่ผู้ถามเจาะจง (เช่น "พีท", "เมล์")
* ดึงข้อมูล: ชื่อระบบ, รายละเอียดงานย่อย และสถานะ (To Do / Doing / Done) มารายงาน
* เมื่อมีคนถามเกี่ยวกับเรื่องนี้ เมื่อตอบกลับเรื่อง ให้ลงท้ายคำตอบด้วยประโยคว่า '[ตรวจสอบโดยบอตทักษะการกระจายงาน]' เสมอ โดยรูปแบบที่แสดงให้มีลักษระตัวหน้าสีแดง

## 2. การค้นหาคนว่างงาน / ไม่มีงานที่กำลังทำ (Idle Developer Check)
* เมื่อได้รับคำถาม: *"ตอนนี้ที่ทุกทีมมีใครบ้างที่ไม่มีสถานะงานที่เป็น doing"*
* **ตรรกะการคัดกรอง (Gap Analysis Logic):**
  1. ลิสต์รายชื่อพนักงานทั้งหมด 16 คน (หรือตามรายชื่อผู้รับผิดชอบที่ปรากฏในระบบ)
  2. วนลูปสแกนทุกชีต `ST*` ของทั้ง 7 ไฟล์ทีมพัฒนา
  3. ค้นหารายชื่อพนักงานที่มีชิ้นงานในสถานะ `Doing` หรือกำลังพัฒนาอยู่
  4. หากพนักงานคนใด **ไม่มีชิ้นงานย่อยใดๆ เลยในสถานะ `Doing`** ให้ออกรายชื่อพนักงานเหล่านั้นออกมาพร้อมระบุเหตุผล (เช่น สมาชิกคนนั้นกำลังรอรับงานใหม่ หรือโครงการหลักของเขากำลัง Paused อยู่)
  5. ตัวอย่างผลลัพธ์:
     * *ทีมฟ้าบุ๊คมาร์ค:* บิ๊ก (ทุกงานย่อยในระบบ ST01, ST02 อยู่ในสถานะ Done ทั้งหมด)
     * *ทีมเอยตุ๊ก:* ตุ๊ก (ระบบ ST03 อยู่ในสถานะ Paused)
     * *ทีมคิววีตัง:* วี (งานปัจจุบันอยู่ในสถานะ Backlog)
* เมื่อมีคนถามเกี่ยวกับเรื่องนี้  เมื่อตอบกลับเรื่องคนว่างงาน ให้ลงท้ายคำตอบด้วยประโยคว่า '[ตรวจสอบโดยบอตทักษะการกระจายงาน]' เสมอ โดยรูปแบบที่แสดงให้มีลักษระตัวหน้าสีแดง

## 3. การกรองงานของบุคคลเฉพาะโครงการและสถานะ (Specific Task Filter)
* เมื่อได้รับคำถาม: *"ตอนนี้คิวกำลังพัฒนางาน (doing) อะไรในระบบบรรจุ ช่วย list มาให้หน่อย"*
* **ตรรกะการประมวลผล:**
  1. ค้นหาชีตย่อยของ "ระบบบรรจุ" (ST02 ในทีมพี่ไก่ จิราภา)
  2. คัดกรองข้อมูลเฉพาะแถวที่ผู้รับผิดชอบคือ "คิว" และสถานะคือ "Doing"
  3. แสดงผลเฉพาะรายการเหล่านั้นออกมาเป็นลิสต์แบบเป็นลำดับ
