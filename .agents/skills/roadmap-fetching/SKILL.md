# Skill: Roadmap Fetching

ทักษะนี้มีไว้เพื่อให้เอเจนต์เข้าใจโครงสร้างและวิธีการดึงข้อมูลจากตาราง Google Sheets ของทั้ง 7 ทีม โดยครอบคลุมชีต 2 ประเภทหลัก:

## 1. การคัดกรองชื่อชีต (Sheet Name Filtering)
* ในแต่ละไฟล์ Google Sheets/Excel จะประกอบไปด้วยหลายชีตย่อย
* เอเจนต์ต้องเลือกวิเคราะห์เฉพาะชีตที่มีความหมายในการทำงานของระบบ:
  1. ชีตที่ชื่อว่า **"Project"** (ชีตสรุปภาพรวมโครงการ)
  2. ชีตที่ชื่อ **ขึ้นต้นด้วยคำว่า "ST" และตามด้วยรหัสตัวเลข** (เช่น `ST1`, `ST2`, `ST3`...) เพื่อดูรายละเอียดงานย่อยของแต่ละโครงการ

## 2. โครงสร้างชีต "Project" (ตารางสรุปภาพรวม)
ข้อมูลแถวแรกจะเป็นหัวตารางเสมอ โดยมีคอลัมน์สำคัญดังนี้:
* **คอลัมน์ B:** รหัสโครงการ (Project ID / ST Code) เช่น ST1, ST2
* **คอลัมน์ C:** ชื่อระบบ (System Name)
* **คอลัมน์ E:** หัวหน้าทีม (Team Leader)
* **คอลัมน์ F:** ผู้ร่วมทีม (Team Members)
* **คอลัมน์ H:** วันที่เริ่มต้น (Start Date)
* **คอลัมน์ I:** วันที่สิ้นสุด (End Date)
* **คอลัมน์ J:** กำหนดส่งงาน (Deadline)
* **คอลัมน์ K:** สถานะของ Project (Status เช่น Doing, Done, Backlog, Paused)
* **คอลัมน์ L:** จำนวนงานย่อยที่เสร็จแล้ว (Done Tasks Count)
* **คอลัมน์ M:** จำนวนงานย่อยทั้งหมด (Total Tasks Count)
* **คอลัมน์ N:** เปอร์เซ็นต์ความคืบหน้าของ Project (Progress %)
* **คอลัมน์ P:** หมายเหตุ (Remarks)

## 3. โครงสร้างชีตรหัส "ST" (ตารางรายละเอียดงานย่อย)
เป็นตารางแสดงงานย่อย (Tasks) ที่ต้องทำในโครงการรหัสย่อยนั้นๆ ประกอบไปด้วย:
* **งานย่อย/กิจกรรม:** คำอธิบายรายละเอียดงานย่อยย่อย
* **ผู้รับผิดชอบ:** ชื่อพนักงานที่รับผิดชอบงานชิ้นนี้ (จาก 16 คนในทีม)
* **สถานะ:** สถานะรายชิ้นงาน ได้แก่ `To Do`, `Doing`, `Done`
* **หมายเหตุ:** ปัญหาอุปสรรค, ความเคลื่อนไหว เช่น ปัจจัยภัยธรรมชาติเลื่อนเดดไลน์

## 4. กฎการเรียกใช้เครื่องมือ (Tool Usage Rules)
* เมื่อผู้ใช้สั่งงาน ให้นึกถึงประเภทคำสั่งว่าต้องการสืบค้นข้อมูลระดับ Macro (ใช้ชีต Project) หรือระดับ Micro (ใช้ชีต ST*) จากนั้นเรียกใช้งานเครื่องมือผ่าน MCP Server:
  * เพื่อสอบถามภาพรวม ความคืบหน้า หรือกำหนดส่ง $\rightarrow$ เรียกใช้เครื่องมืออ่านชีต **Project**
  * เพื่อเจาะลึกงานย่อยหรือการติดตามงานบุคคล $\rightarrow$ เรียกใช้เครื่องมืออ่านชีต **ST***
