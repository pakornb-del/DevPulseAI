# Skill: Weekly Report Summarizer

ทักษะนี้มีไว้สำหรับสรุปผลการปฏิบัติงานรายสัปดาห์ เพื่อเตรียมส่งต่อรายงานให้ผู้บริหารหรือใช้สำหรับการประชุมสรุปงาน (Weekly Standup):

## 1. ข้อมูลที่ต้องสรุป (Key Metrics to Extract)
* **ความคืบหน้าภาพรวม (Overall Progress):** ตรวจสอบจำนวนโครงการทั้งหมดข้ามทั้ง 7 ชีตเปรียบเทียบระหว่าง โครงการที่เสร็จสิ้น (`Done`), โครงการที่กำลังทำอยู่ (`Doing`), โครงการที่ถูกระงับ (`Paused`), และโครงการที่ค้างในคิว (`Backlog`)
* **ผลงานเด่นประจำสัปดาห์ (Key Achievements):** รายการงานย่อยที่เปลี่ยนสถานะเป็น `Done` ในช่วงสัปดาห์ล่าสุด
* **คอขวดที่ตรวจพบ (Blocked / Paused Projects):** รายการโครงการย่อยที่มีสถานะเป็น `Paused` หรือ `Backlog` หรือหมายเหตุสำคัญที่ระบุอุปสรรค (เช่น ย้ายงานไปปีงบ 70 หรือน้ำท่วมภาคใต้เลื่อนวันรับสมัคร)

## 2. โครงสร้างรายงานสรุป (Executive Summary Structure)
เมื่อผู้ใช้สั่งให้สรุปผลประจำสัปดาห์ เอเจนต์ควรนำเสนอข้อมูลในรูปแบบข้อความ Markdown ที่อ่านง่ายและแบ่งหัวข้อชัดเจนดังนี้:

```markdown
### 📊 รายงานภาพรวมแผนงานประจำสัปดาห์ (สรุปจาก 7 ทีมพัฒนา)

**1. สถานะโครงการภาพรวม:**
* โครงการทั้งหมด: [จำนวน] โครงการ
* เสร็จสิ้นแล้ว (Done): [จำนวน] โครงการ
* กำลังพัฒนาอยู่ (Doing): [จำนวน] โครงการ
* หยุดชะงักชั่วคราว (Paused): [จำนวน] โครงการ
* รอดำเนินการ (Backlog): [จำนวน] โครงการ

**2. ประเด็นติดขัดและคอขวดที่ต้องการการดูแล (Blockers & Paused):**
* **ทีมไก่:** ระบบงบประมาณคำขอเงินรายได้ และระบบงบประมาณเงินอุดหนุนรัฐบาล (สถานะ Paused - ย้ายไปปีงบ 70)
* **ทีมสุ_สา_แก๊ป:** ระบบรับสมัคร TCAS69 (ได้รับผลกระทบจากอุทกภัยภาคใต้ ปรับกำหนดการส่งมอบ)
* **ทีมคิววีตัง:** ระบบ TQF ปรับปรุงระบบ (สถานะ Backlog)

**3. รายชื่อพนักงานที่คิวงานว่างชั่วคราวในสัปดาห์นี้:**
* [รายชื่อพนักงานพร้อมระบุทีมและเหตุผล]
```
